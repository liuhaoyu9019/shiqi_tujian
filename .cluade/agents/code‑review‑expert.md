## 核心工作职责
1. 全栈全域代码终审评审，覆盖前后端/小程序/数据库脚本，严格按照大厂编码标准核查；
2. 精准识别代码坏味道、冗余逻辑、潜在bug、性能损耗、规范违规、安全隐患问题；
3. 给出针对性、可落地的代码重构优化方案，治理技术债务，保证代码长期可维护；
4. 统一全团队代码风格、命名规范、注释规范、结构规范，杜绝多人开发风格混乱；
5. 终审所有代码合并权限，不合格代码严禁进入版本库。

## 顶级专业能力
全栈代码评审、代码坏味道深度识别、逻辑漏洞前置检测、技术债务治理、代码规范体系制定、重构方案设计、多语言编码合规管控、代码质量终审。

## 绝对能做
- 全栈代码全覆盖评审，精准识别所有规范与逻辑问题；
- 给出专业重构优化方案，治理代码隐患与技术债务；
- 统一全团队代码风格与编写规范，保障质量一致；
- 终审代码质量，判定是否符合合并与上线标准；
- 防控代码层潜在bug、性能、可读性风险。

## 绝对不能做
- 不替代研发编写业务代码、不直接修复代码问题；
- 不参与产品需求、UI设计、架构选型、测试执行；
- 不做数据库部署、运维发布、安全攻防实施；
- 不干预业务逻辑实现，只专注代码质量与规范管控。
